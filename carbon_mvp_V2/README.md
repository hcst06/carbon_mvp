# 碳行益农 - 基于绿色出行的碳积分助农平台

## 项目简介

碳行益农是一个基于绿色出行的碳积分助农平台，通过记录用户的绿色出行行为（如步行、骑行、公交、地铁）计算碳减排量，并将其转化为积分，用户可以使用这些积分兑换农产品，从而实现绿色出行与乡村振兴的良性循环。

## 文件作用详解

### 后端文件

#### 1. backend/app.py
- **作用**：后端主应用文件，实现了所有API接口和数据库操作
- **重点代码**：
  - `init_db()`: 初始化数据库，创建用户表、记录表、产品表和兑换记录表
  - `register()`: 用户注册接口，处理用户注册逻辑
  - `login()`: 用户登录接口，验证用户身份
  - `calculate()`: 碳积分计算接口，处理碳减排计算和区块链存证
  - `get_user_stats()`: 用户统计数据接口，获取用户的碳减排统计信息

#### 2. backend/calculator.py
- **作用**：碳减排计算模块，根据出行方式和距离计算碳减排量
- **核心逻辑**：
  - 定义了不同出行方式的碳排放系数
  - `calculate_carbon(distance, mode)`: 根据距离和出行方式计算碳减排量

#### 3. backend/anti_cheat.py
- **作用**：反作弊系统，检测用户提交的出行数据是否合理
- **核心逻辑**：
  - 定义了不同出行方式的合理速度范围
  - `check_cheat(distance, time, mode)`: 检测数据是否异常

#### 4. backend/blockchain.py
- **作用**：区块链存证模块，将碳积分数据存储到区块链中
- **核心逻辑**：
  - `Blockchain` 类：实现区块链的基本功能
  - `create_carbon_block()`: 为碳积分记录创建区块
  - `calculate_hash()`: 计算区块的哈希值

## 技术原理

### 1. 碳积分计算原理
- **计算方法**：根据出行距离和出行方式，使用碳排放系数计算碳减排量
- **积分转换**：将碳减排量转换为积分，100g碳减排 = 1积分
- **出行方式系数**：
  - 步行：0 g/km
  - 骑行：0 g/km
  - 公交：89 g/km
  - 地铁：41 g/km

### 2. 区块链存证原理
- **区块结构**：每个区块包含索引、时间戳、数据、前一个区块的哈希值和当前区块的哈希值
- **哈希计算**：使用SHA-256算法计算区块哈希值
- **链式结构**：每个区块通过哈希值链接到前一个区块，形成不可篡改的链式结构
- **存证内容**：记录用户ID、出行距离、出行方式、时间、减排量和积分

### 3. 反作弊原理
- **速度检测**：根据距离和时间计算速度，判断是否在合理范围内
- **出行方式验证**：为不同出行方式设置合理的速度范围
- **数据异常处理**：当检测到异常数据时，拒绝计算并提示用户

### 4. 数据存储原理
- **数据库**：使用SQLite数据库存储用户数据、出行记录、产品信息和兑换记录
- **表结构**：
  - `users`: 存储用户信息
  - `records`: 存储出行记录
  - `products`: 存储产品信息
  - `exchanges`: 存储兑换记录
- **本地存储**：小程序使用wx.setStorageSync存储用户登录信息
- **缓存机制**：实现请求缓存，提高性能

## 使用方法

### 1. 启动后端服务
```bash
# 安装依赖
pip install flask flask-cors

# 启动服务
python backend\app.py
```

### 2. 运行微信小程序
1. 打开微信开发者工具
2. 导入 `miniprogram` 目录
3. 勾选"不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书"
4. 点击"编译"运行小程序

### 3. 功能使用
- **注册/登录**：使用账号密码注册或微信登录
- **计算碳积分**：填写出行距离、时间，选择出行方式，点击"计算碳积分"
- **出行轨迹**：在"轨迹"页面开始记录绿色出行轨迹，系统会自动计算距离和碳积分
- **查看记录**：在"记录"页面查看历史出行记录
- **积分兑换**：在"兑换"页面使用积分兑换农产品
- **查看统计**：在"统计"页面查看碳减排统计数据和图表
- **成就系统**：在"成就"页面查看成就和任务，完成任务获得额外积分
- **分享功能**：在首页点击分享按钮，分享给好友或朋友圈

## 数据存储说明

### 1. 数据库结构
- **users表**：
  - id: 用户ID
  - username: 用户名
  - password: 密码
  - email: 邮箱
  - total_points: 总积分
  - created_at: 创建时间

- **records表**：
  - id: 记录ID
  - user_id: 用户ID
  - distance: 距离（km）
  - mode: 出行方式
  - time: 时间（分钟）
  - reduction: 减排量（g）
  - points: 获得积分
  - created_at: 创建时间

- **products表**：
  - id: 产品ID
  - name: 产品名称
  - description: 产品描述
  - price_points: 兑换积分
  - stock: 库存
  - image_url: 图片URL

- **exchanges表**：
  - id: 兑换ID
  - user_id: 用户ID
  - product_id: 产品ID
  - points: 消耗积分
  - created_at: 创建时间

### 2. 区块链存储
- **区块数据**：存储在内存中，重启服务后会重置
- **持久化**：可扩展为将区块链数据存储到文件或数据库中
- **验证机制**：通过哈希值验证区块的完整性

### 3. 本地存储
- **用户信息**：使用wx.setStorageSync存储用户登录信息
- **缓存数据**：缓存常用数据，提高小程序响应速度
- **轨迹数据**：临时存储出行轨迹数据

## 注意事项

1. **开发环境**：
   - 后端服务运行在 `http://127.0.0.1:5000`
   - 微信开发者工具需要勾选"不校验合法域名"

2. **生产环境**：
   - 需要部署到支持HTTPS的服务器
   - 需要在微信小程序后台配置合法域名
   - 需要实现真实的微信登录功能

3. **数据安全**：
   - 密码存储建议使用加密算法
   - 敏感数据传输建议使用HTTPS
   - 定期备份数据库

4. **性能优化**：
   - 优化数据库查询，使用索引
   - 缓存热点数据
   - 减少网络请求次数
   - 图片优化，使用WebP格式图片
   - 代码分包，减少初始加载时间
